2. 技术架构与关键约束

2.1 为什么不能"前端直连 API"

  • CORS：Yahoo、东方财富的行情接口不返回 Access-Control-Allow-Origin，浏览器跨域请求会被直接拦截（实测东方财富 push2his 无 CORS 头）。
  • 反爬 / 限流：上述接口按源 IP 限流，浏览器暴露的是用户 IP，且频繁请求会被临时封禁；GitHub Pages 的全球 CDN 边缘节点 IP 更不可控。
  • 密钥暴露：FRED / Tushare 需要 Key，若在前端硬编码等于公开泄露，被他人盗刷配额。
架构铁律：数据获取与计算必须 100% 发生在服务端（GitHub Actions Runner / 自建服务器），前端只消费静态 JSON。这同时带来一个好处——前端极轻、秒开、可缓存、可被任意 CDN 加速。

2.2 "自动化 Agent 能力"的两种落地形态

需求要求"定时触发 Agent 脚本自动更新"。存在两种等价落地形态，二者共用同一份 fetch_data.py，仅调度器不同：

形态调度位置密钥存放适用
A. WorkBuddy 本地自动化（首选）本机 WorkBuddy 运行时，按 RRULE 定时唤醒 Agent 执行脚本本地环境变量 / 本地 .env（gitignore），不进仓库、不进日志隐私优先、密钥不出本机、私有/本地协同
B. GitHub Actions（云端替补）GitHub Runner，cron 触发GitHub Secrets（加密，但公共仓库若误打日志有泄露风险）公开站点、需云端 24/7 无人值守
经本轮重评估，首选形态 A（WorkBuddy 本地自动化）：用户核心诉求是"密钥在本地、不容易泄露"，本地自动化天然满足——密钥永不离开本机，且脚本运行上下文即用户本地环境。代价是调度依赖本机/应用在线（详见第 11 章权衡）。若需"全球任意时刻无人值守"，再叠加形态 B。

5. 牛熊判定算法规范

对每个价格序列计算三类技术子分，加权得到序列分；序列分聚合为市场维度分；维度分聚合为综合结论。

5.1 均线排列分 ma_score ∈ [-1,1]

  • 多头排列（价格 > MA20 > MA60 > MA200）= +1；空头排列 = -1。
  • 中间态按成立条件数线性映射：ma_score = (成立数/3)×2 − 1。
  • MA 窗口：MA20（短期趋势）、MA60（中期）、MA200（长期牛熊分界，经典"年线"）。

5.2 RSI(14) 分

rsi_score = (RSI − 50) / 50，并作 [-1,1] 截断。>50 偏多，<50 偏空；极端值（>70 / <30）本身也提示反转风险，可在 UI 标注。

5.3 MACD(12,26,9) 分

取柱状图（DIF−DEA）相对价格归一：macd_score = clamp( hist / (2% × 价格) , -1, 1)。红柱扩张=多头动能，绿柱扩张=空头动能；金叉/死叉可作离散事件标记。

5.4 市场广度（进阶，建议二期）

真正的"市场宽度"需成分股数据（上涨/下跌家数、创52周新高家数等）。免费近似：① 用宽基指数(沪深300/标普500)替代；② 用 stocks_above_ma 代理——抓取指数成分股中站上 MA200 的比例。无免费成批接口时，可暂以"指数本身 + 恐贪"代理广度，二期接入 Tushare 成分股。

5.5 恐惧贪婪指数分

fng_score = (FNG − 50) / 50，0–100 映射为 [-1,1]，作为情绪维度直接并入美股综合分。

5.6 债券市场的反向处理

债券"牛熊"与收益率反向：收益率上行=债券价格下跌=债市走熊。因此对债券收益率序列取 score = −composite(收益率)。

5.7 加权与阈值

层级公式
序列分0.5·MA + 0.25·RSI + 0.25·MACD
市场维度分该维度下各序列分（含情绪）等权平均
综合结论各维度分等权平均
判定阈值≥ +0.33 牛市 · ≤ −0.33 熊市 · 之间震荡
权重为可解释基线，建议在 UI 提供"权重配置"面板，让用户按偏好微调（如激进者提高 MACD 权重）。所有分数均以 [-1,1] 存储，便于热力图统一着色。

11. 自动化形态重评估：WorkBuddy 本地自动化

用户明确选择"密钥在本地、不容易泄露"，并采用 WorkBuddy 自动化。本章据此重评估调度形态，给出推荐架构与诚实权衡。

11.1 工作机制

WorkBuddy 自动化任务持久化于本地 ~/.workbuddy/workbuddy.db，按 RRULE（如每个工作日 0 21 * * 1-5）定时唤醒本机 Agent，执行一段自然语言 prompt：运行 fetch_data.py → 校验 market.json → 可选推送到 GitHub Pages。脚本运行在用户本机环境，密钥从本地环境变量或本地 .env（已 gitignore）读取，永不进入仓库、永不出现在任何远端日志。

11.2 密钥安全对比（本地 .env vs GitHub Secrets）

维度WorkBuddy 本地自动化（本地 .env）GitHub Actions（Secrets）
密钥落点本机文件，仅本机可读GitHub 加密存储（仓库级/组织级）
是否入库/入远端否 gitignore 排除存储在 GitHub 云端
日志泄露风险极低 本地运行，无远端日志中 公共仓库若 echo $KEY 或第三方 Action 读取会泄露
泄露面本机被入侵（与操作系统同级别）GitHub 账号/Runner 供应链
调试直观度高 本地即时看输出中 需查 Actions 日志
结论：在"密钥不泄露"这一诉求上，本地自动化是更优解——密钥的物理边界就是用户本机，不存在"公共仓库误提交/误打印"这类典型泄露路径。GitHub Secrets 本身是安全的，但公共仓库场景下维护者的人为失误（日志打印、fork 后误配）是真实风险点，而本地方案从结构上规避了它。

11.3 诚实的权衡

维度WorkBuddy 本地自动化GitHub Actions
24/7 无人值守中 依赖本机/WorkBuddy 在线；关机则错过该次，下次补跑稳 云端常驻
频率RRULE 支持 DAILY/WEEKLY 等，满足日/周更cron 最小 5 分钟
跨设备/协作默认仅本机天然多端、可协作
发布到公网需自动化内显式 git push 或手动发布原生集成 Pages 部署
关键提醒：本地自动化"依赖本机在线"是真实约束。缓解：(1) 配置 WorkBuddy/本机开机自启；(2) 采用双轨——本地自动化为主、GitHub Actions 为备（仅当连续 N 天无更新才触发），兼顾"密钥本地"与"云端不中断"；(3) 即便错过数次，页面渲染的是上一次成功 JSON（降级策略见第 7 章），不会"开天窗"。

12.4 回测一致性校验（算法是否含信息？）

专业分析必须回答一个质疑：这套算法是真的在"判断"，还是只是在拟合噪声？为此新增 scripts/backtest.py，复用 fetch_data.py 的纯函数，对标的做"滚动判定"——在每个交易日用当时可得的历史算出牛熊分与结论，再考察其后 N 日的前瞻收益，统计各结论态下的命中率与"分↔前瞻收益"相关系数。

诚实解读（避免夸大）：
  • 状态判别有效：算法能稳定区分"当前处于牛机制"——牛态标签下 21 日上涨概率 66%、252 日 98.7%，显著高于随机，说明它作为机制/状态标签是可靠的。
  • 分数与前瞻收益负相关（两种周期皆然）：最高分的"极度看涨"读数往往出现在动量透支后的局部高位，随后更易出现均值回归——这是成熟市场经典的动量衰减/短期反转现象，而非算法缺陷。
  • 因此正确用途：本分数是"现在处于什么机制"的标签，而非"未来 N 日涨跌"的择时信号；若要做择时，应叠加估值（CAPE）与范式相位（狂热期对牛态标签降权）做二次过滤——这恰是 13 章二维判读框架的用武之地。
  • 样本偏差已闭合：单窗口（2023–2026）本身强多头，命中率被 regime 抬高；--multi 样本外测试扩展至 2014–2026，覆盖 2018 抛售、2020 新冠崩盘、2022 熊市，合计 3550 个决策点，牛态命中率仍 66.2%（>50%）且分↔前瞻收益持续负相关（−0.115），证明算法在不同机制下均稳定含信息，而非多头市特例。
  • 熊态标签的前瞻收益为正：样本外"熊"标签平均 21 日前瞻 +2.59%、命中 65.4%——因熊态标签多聚集在急跌末端，随后均值回归反弹。再次印证本分数宜作"机制标签 + 风险温度计"，而非反向做空信号。

13. 《技术革命与金融资本》范式框架的引入

为弥补"结构层"缺失并提升方法论深度，引入 Carlota Pérez《技术革命与金融资本》（2002 英文版；中译 2007 中国人民大学出版社；2025 湛庐再版，新增 AI 洞察）的"技术-经济范式"模型。该框架解释了为何金融泡沫是技术革命的常态而非异常，正好克制当前算法"在狂热期被价格趋势强烈向上扭曲"的盲区。

13.1 核心模型

  • 五次技术革命（约每 50–60 年一次）：①1771 工业革命（水力/棉纺）②1829 蒸汽与铁路 ③1875 钢铁/电力/重工程 ④1908 石油/汽车/大规模生产 ⑤1971 信息与电信；当前正处于第六次"数字与智能革命"（AI）。
  • 两大时期：导入期（Installation）——新技术闯入、基础设施被金融资本狂热建设；展开期（Deployment）——泡沫破裂后制度适配、生产资本接管、进入"黄金时代"。
  • 四/五阶段：爆发（Irruption）→ 狂热（Frenzy）→ 转折点（Turning Point & 崩溃）→ 协同（Synergy）→ 成熟（Maturity）。
  • 两类资本：金融资本（追逐叙事与纸面增值、高流动性、易投机）vs 生产资本（关心可持续利润、与实体耦合）。泡沫源于二者"脱钩"，黄金时代源于二者"再耦合（recoupling）"。

13.2 当前 AI 革命的定位（综合 2025 年多方研究）

13.4 落地为仪表盘模块

新增"技术-经济范式周期"卡片：展示五次革命时间轴 + 当前所处阶段标注 + 对当前市场的含义；范式阶段为低频（季度）人工/半自动复核，存于 data/paradigm.json（避免被日频噪声干扰）。该模块让仪表盘从"只看价格趋势"升级为"价格趋势 + 50 年结构周期"的双坐标判读。
